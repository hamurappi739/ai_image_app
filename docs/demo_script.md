# Demo Script — AI Image App MVP

Сценарий демонстрации текущего MVP для презентации руководству и заинтересованным сторонам.

---

## 1. Цель демонстрации

Коротко объяснить аудитории:

- Это **текущий MVP** — рабочий прототип продукта, а не финальный релиз.
- **Реальная генерация** (Gemini), **авторизация**, **загрузка фото** и **оплата** будут подключены **следующими этапами**.
- Сейчас важно показать **пользовательский flow** и **продуктовую логику**: как человек без технического бэкграунда создаёт изображения, выбирает фотосессии и видит результат в галерее.

---

## 2. Что уже работает

| Область | Что есть в MVP |
|---------|----------------|
| **Создать** | Ввод описания, подсказки, быстрые идеи, генерация через backend `POST /generate` |
| **Результат** | Изображение на экране после генерации (в demo-mode — fallback-preview) |
| **Галерея** | Кнопка **«Открыть в Галерее»**; загрузка истории через `GET /generations` |
| **Галерея** | Новые изображения появляются **сверху**; локальная кнопка **«Очистить»** |
| **Фотосессии** | 8 готовых стилей (3 бесплатных, 5 по 100 ₽) |
| **Фотосессии** | Demo modal (bottom sheet) будущей **загрузки фото** |
| **Пакеты** | Экран пакетов генераций (199 / 499 / 1199 ₽), без реальной оплаты |
| **Профиль** | Placeholder под будущий вход и настройки |

**Сказать:** «Это простой интерфейс для пользователя без технических терминов.»

- Убедиться, что backend запущен (`uvicorn` на порту 8000).

**Ввести пример:**

**Сказать:** «Очистка сейчас **локальная** — только на этом устройстве в текущей сессии. Она **не удаляет** серверную историю в Supabase. Это удобно для демонстраций и тестов.»

### Шаг 6. Показать «Фотосессии»

**Сказать:** «Это **основной простой сценарий** для массовой аудитории: выбрать готовый стиль, загрузить своё фото и получить **3 изображения в одной теме**.»

**Сказать:** «**Авторизация** будет добавлена позже. После входа здесь будут сохраняться изображения, фотосессии и покупки. Секретные ключи и платежи не хранятся в приложении — важные операции на сервере.»

## 4. Что важно подчеркнуть

- Интерфейс на **русском** языке.
- В пользовательском UI **нет** слов «промпт», «кредиты», «токены» — только **описание**, **генерации**, **пакеты**, **фотосессии**.
- **Основной массовый сценарий** — **Фотосессии** (готовый стиль + своё фото).
- **Платные** фотосессии (100 ₽) должны запускаться **после оплаты**, не до неё.
- Backend уже готовит **историю** через Supabase (`generations`, `GET /generations`).
- **`/debug/*` endpoints** — только для разработки; **перед релизом** их нужно удалить или защитить.

## 5. Следующие этапы

1. Подключить **реальную Gemini-генерацию**.
2. Добавить **загрузку пользовательского фото** для фотосессий.
3. Добавить **обработку фото** на backend (3 изображения в стиле).
4. Добавить **авторизацию** (Supabase Auth).
5. Сохранять **историю по аккаунту** на всех устройствах.
6. Подключить **RuStore Billing** (пакеты и фотосессии).
7. **Закрыть или защитить** debug endpoints.
8. Подготовить **production config** (CORS, секреты, RLS).
