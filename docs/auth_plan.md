# План авторизации

Текущее состояние входа в аккаунт в приложении.

## Что работает сейчас

### Вход по почте

- Реализован через **Supabase Auth** на стороне клиента (Flutter).
- Работает только если при сборке заданы `SUPABASE_URL` и `SUPABASE_ANON_KEY` (`--dart-define`).
- На экране **Профиль** кнопка **«Войти по почте»** открывает форму:
  - поля Email и Пароль;
  - **«Войти»** — вход в существующий аккаунт;
  - **«Создать аккаунт»** — регистрация.
- После успешного входа приложение передаёт сессию в API (Bearer) для баланса, галереи и генерации.
- Ошибки показываются простым языком, без технических деталей.

### Демо-режим (без auth config)

- Если Supabase не настроен, профиль показывает **«Демо-режим»**.
- Вход по почте недоступен — показывается понятный dialog.
- Баланс и функции приложения работают в режиме демо (как и раньше).

## Что пока только в UI (заглушки)

### VK ID

- Кнопка **«Продолжить с VK ID»** на экране профиля.
- Не запускает OAuth.
- Показывает dialog: «VK ID скоро появится».

### Яндекс ID

- Кнопка **«Продолжить с Яндекс ID»**.
- Не запускает OAuth.
- Показывает dialog: «Яндекс ID скоро появится».

## Следующий этап: реальные провайдеры

Для подключения VK ID и Яндекс ID потребуется отдельная настройка:

1. **Supabase Dashboard** — включить OAuth-провайдеры (VK, Yandex) или custom OIDC.
2. **Redirect URLs** — зарегистрировать callback для мобильного приложения (deep link / custom scheme).
3. **Flutter** — вызов `signInWithOAuth` (или аналог) вместо заглушек; обработка возврата в приложение.
4. **Юридическое** — политика конфиденциальности и согласие на обработку данных для соцсетей.

Восстановление пароля («Забыли пароль?») — отдельная задача через Supabase `resetPasswordForEmail`.
