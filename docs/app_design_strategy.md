# App Design Strategy

Концепция дизайна и структуры **AI Image Generator** для мобильного приложения (Flutter).

**Аудитория:** 30+, в том числе **40–60+** — простой, понятный, «дорогой» интерфейс.  
**Идея продукта:** не «писать промпты», а **выбрать готовую фотосессию** или описать идею своими словами.  
**Язык UI (текущий MVP):** **русский**.

---

## 1. Design direction

| Принцип | Реализация |
|---------|------------|
| Стиль | Светлый **премиальный** UI: много воздуха, мягкие тени, нейтральный фон |
| Карточки | **Крупные** preview-карточки шаблонов, легко нажимаются пальцем |
| Действия | Понятные кнопки: «Хочу так же», «Создать», «Скачать» |
| Язык | **Минимум** технических терминов |
| Запрет в UI | **«промпт»**, **«кредиты»**, **«токены»**, **credits**, **tokens** |

**В интерфейсе использовать:**

- **генерации**
- **бесплатные генерации**
- **пакеты генераций**
- **фотосессии**
- **осталось генераций** (вместо paid credits)

**Вместо «prompt»:** «опишите изображение», «идея», «описание».

Технические `prompt`, `paid_credits`, `credit_consumed` — только backend / JSON (см. [api_contract.md](api_contract.md)).

## 2. Main user idea

Пользователь видит **красивый пример** и хочет **похожее фото себя**.

### Главный сценарий (целевой)

1. Выбрать **шаблон** (портрет или фотосессию)
2. **Загрузить своё фото** (лицо / портрет)
3. Нажать **«Создать»**
4. Получить **результат**

**MVP:** загрузка фото — **mock / placeholder** (иконка «добавить фото», без реального upload). Реальная загрузка и face-swap — позже.

## 3. Bottom navigation (реализовано во Flutter)

Нижнее меню — **5 вкладок**, подписи на **русском**:

| Вкладка | EN (код) | Назначение | Backend |
|---------|----------|------------|---------|
| **Создать** | Create | Свой текстовый запрос → одно изображение | `POST /generate` |
| **Фотосессии** | Photoshoots | Готовые наборы по **3 фото**, free / 100 ₽ | Пока UI-заглушка |
| **Галерея** | Gallery | Созданные изображения пользователя | Placeholder |
| **Пакеты** | Packs | Пакеты генераций (199 / 499 / 1199 ₽) | Placeholder, оплата позже |
| **Профиль** | Profile | Настройки, аккаунт (позже) | Placeholder |

`IndexedStack` сохраняет состояние вкладки **Создать** при переключении.

## 4. Portrait templates (позже)

Отдельный каталог **портретов** (1 фото) возможен после MVP. Сейчас во Flutter объединено с **Фотосессиями** (карточки «3 фото»).

### Структура

- **10–15** шаблонов на старте (mock в MVP)
- У каждого: **иконка / preview**, название, короткий подзаголовок
- Внутри шаблона — **заготовленный prompt** (скрыт от пользователя)
- CTA: **«Хочу так же»** → экран создания (фото + «Создать»)

### Динамика (после MVP)

Шаблоны с backend/CMS: добавлять тренды, скрывать устаревшие, A/B порядок карточек.

### Примеры шаблонов

- Человек с тигром  
- Деловой портрет  
- Фото на яхте  
- Портрет в стиле журнала  
- В поле с цветами  
- Вечерний город  
- Премиальный интерьер

## 5. Photoshoot mode (вкладка «Фотосессии»)

Раздел **«Фотосессии»** во Flutter.

### Ощущение продукта

Одна **тема**, одна **локация**, одна **атмосфера** — **3 фото** одного человека с **разных ракурсов**, как от одного фотографа.

**Пример:** «Фотосессия в поле с пионами» — три кадра: анфас, три четверти, по пояс; одинаковый свет и фон.

### MVP credit hypothesis (не финально)

| Действие | Генерации (гипотеза) |
|----------|----------------------|
| Обычная генерация (1 фото) | 1 |
| Фотосессия (3 фото) | 3 |
| Бесплатный preview с watermark | 1 раз (на пользователя) |
| Скачать без watermark | после оплаты / пакета |

> Цены и экономика — **уточняются** после тестов (см. [product_strategy.md](product_strategy.md)). В UI не писать «кредиты».

**MVP UI:** mock-карточки фотосессий без реальной генерации трёх изображений.

## 6. Gallery (Галерея)

Раздел **«Галерея»** (бывш. History / «Моё» в ранних планах).

- **Библиотека** ранее созданных изображений (сетка или список)
- MVP: заглушка или данные с `GET` истории (когда появится публичный API)

**Позже:**

- скачать  
- повторить стиль  
- удалить  
- «создать похожее»  

---

## 7. Packs (Пакеты)

Вкладка **«Пакеты»** — пакеты **генераций** для custom mode (Стартовый / Авторский / Профи).  
MVP: UI-заглушка, кнопка «Скоро», без RuStore. Не путать с оплатой **фотосессии** (100 ₽ за набор 3 фото).

---

## 8. Create (Создать)

Вкладка **«Создать»** — custom mode.

Пользователь **описывает словами**, что хочет получить (поле «Например: портрет в деловом стиле»).

Переключатель режима:

- **Одно фото**
- **Фотосессия** (3 фото — позже)

В UI — только «идея» / «описание», **не** «промпт».

MVP: одно поле + кнопка «Создать» → `POST /generate` с текстом пользователя (без шаблона).

## 9. MVP scope for UI

### Сделано в текущем Flutter MVP

- [x] Нижняя навигация: **Создать · Фотосессии · Галерея · Пакеты · Профиль**
- [x] **Создать** — `POST /generate`, статус генераций, chips, результат
- [x] **Фотосессии** — 8 карточек (3 free, 5 × 100 ₽), без API
- [x] **Галерея** — пустое состояние
- [x] **Пакеты** — 3 пакета генераций, кнопка «Скоро»
- [x] **Профиль** — placeholder
- [x] Русский UI без «кредитов» / «токенов»

### Не делаем в текущем UI MVP

- Реальная загрузка фото пользователя  
- Watermark / preview vs full quality  
- Настоящая фотосессия из 3 изображений  
- Оплата (RuStore)  
- Сложная библиотека с фильтрами  
- Авторизация

## 10. Future backend implications

Расширения API и БД для шаблонов, фотосессий и загрузок:

| Поле / концепт | Назначение |
|----------------|------------|
| `generation_type` | `single` \| `photoshoot` |
| `template_id` | Связь с каталогом шаблонов |
| `credit_cost` | 1 или 3 (и др.) за операцию |
| `is_watermarked` | Preview vs финальный файл |
| `group_id` | Объединяет 3 фото одной фотосессии |
| `source_image_url` | Фото пользователя для img2img / face |

Таблица `generations` и credit logic потребуют миграций — см. [database_schema.md](database_schema.md).
