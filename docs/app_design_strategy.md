# App Design Strategy

Концепция дизайна и структуры **AI Image Generator** для мобильного приложения (Flutter).

**Аудитория:** 30+, в том числе **40–60+** — простой, понятный, «дорогой» интерфейс.  
**Идея продукта:** не «писать промпты», а **выбрать готовую фотосессию** или описать идею своими словами.  
**Язык UI (текущий MVP):** основной пользовательский интерфейс — **русский** (вкладки, кнопки, подсказки, empty state).

---

## 1. Design direction

| Принцип | Реализация |
|---------|------------|
| Стиль | Светлый **премиальный** UI: фон `#F7F8FC`, белые **мягкие карточки**, лёгкие тени |
| Акцент | **Фиолетово-синий** (`#5B6CFF`, градиенты `#7C5CFF` → `#4A7CFF`) на CTA и выделениях |
| Карточки | Крупные preview-блоки (фотосессии, пакеты), удобный tap-target |
| Действия | Понятные кнопки: «Создать изображение», «Попробовать», «Скоро», «Создать первое изображение» |
| Тексты | Короткие формулировки для аудитории **40–60+**, **минимум** технических терминов |
| Запрет в UI | **«промпт»**, **«кредиты»**, **«токены»**, **credits**, **tokens** |

### UX language rules

**Не использовать в видимом UI:** промпт, кредиты, токены.

**Использовать:**

- **описание** (поле ввода на «Создать»)
- **идея** (быстрые chips «Попробуйте идею»)
- **изображение** / **изображений** (пакеты, галерея, результат)
- **генерации**, **бесплатные генерации**, **купленные генерации**
- **пакеты генераций**, **осталось генераций**
- **фотосессия** / **фотосессии**

Технические `prompt`, `paid_credits`, `credit_consumed` — только backend / JSON (см. [api_contract.md](api_contract.md)).

## 2. Main user idea

Пользователь видит **красивый пример** и хочет **похожее фото себя**.

### Главный сценарий (целевой)

1. Выбрать **шаблон** (портрет или фотосессию)
2. **Загрузить своё фото** (лицо / портрет)
3. Нажать **«Создать»**
4. Получить **результат**

**MVP:** загрузка фото — **mock / placeholder** (иконка «добавить фото», без реального upload). Реальная загрузка и face-swap — позже.

## 3. Bottom navigation (реализовано во Flutter)

Нижнее меню — **5 вкладок**, подписи на **русском**:

| Вкладка | EN (код) | Текущее состояние UI | Backend |
|---------|----------|----------------------|---------|
| **Создать** | Create | Описание + быстрые идеи + подсказки → одно изображение | `POST /generate` |
| **Фотосессии** | Photoshoots | 8 карточек с preview, 3 free / 5 × 100 ₽, без upload и оплаты | UI-заглушка |
| **Галерея** | Gallery | Empty state + переход на «Создать» | Нет API |
| **Пакеты** | Packs | 3 пакета генераций, кнопка «Скоро» | Нет оплаты |
| **Профиль** | Profile | Placeholder входа и будущих настроек | Нет auth |

`IndexedStack` сохраняет состояние вкладки **Создать** при переключении. С **Галереи** кнопка «Создать первое изображение» переключает `selectedIndex` на **Создать** (без backend).

## 4. Portrait templates (позже)

Отдельный каталог **портретов** (1 фото) возможен после MVP. Сейчас во Flutter объединено с **Фотосессиями** (карточки «3 фото»).

### Структура

- **10–15** шаблонов на старте (mock в MVP)
- У каждого: **иконка / preview**, название, короткий подзаголовок
- Внутри шаблона — **заготовленный prompt** (скрыт от пользователя)
- CTA: **«Хочу так же»** → экран создания (фото + «Создать»)

### Динамика (после MVP)

Шаблоны с backend/CMS: добавлять тренды, скрывать устаревшие, A/B порядок карточек.

### Примеры шаблонов

- Человек с тигром  
- Деловой портрет  
- Фото на яхте  
- Портрет в стиле журнала  
- В поле с цветами  
- Вечерний город  
- Премиальный интерьер

## 5. Photoshoot mode (вкладка «Фотосессии»)

Раздел **«Фотосессии»** во Flutter — **главный простой сценарий** для массовой аудитории: выбрать готовый стиль без составления длинного описания.

### Ощущение продукта

Одна **тема**, одна **локация**, одна **атмосфера** — **3 фото в одном стиле** одного человека с **разных ракурсов**, как от одного фотографа.

**Пример:** «Фотосессия в поле с пионами» — три кадра: анфас, три четверти, по пояс; одинаковый свет и фон.

### MVP credit hypothesis (не финально)

| Действие | Генерации (гипотеза) |
|----------|----------------------|
| Обычная генерация (1 фото) | 1 |
| Фотосессия (3 фото) | 3 |
| Бесплатный preview с watermark | 1 раз (на пользователя) |
| Скачать без watermark | после оплаты / пакета |

> Цены и экономика — **уточняются** после тестов (см. [product_strategy.md](product_strategy.md)). В UI не писать «кредиты».

**Текущий Flutter UI (MVP):**

- **8 готовых фотосессий** — основной **простой сценарий** для массовой аудитории (не нужно придумывать описание).
- На карточке: **«3 фото в одном стиле»**, badge **«Бесплатно»** или **«100 ₽»**.
- **Preview-блок** сверху: градиент + инициалы стиля + иконка (без реальных фото).
- **Responsive layout:** 2 колонки на широком web, 1 колонка на узком экране.
- Кнопки **«Попробовать»** / **«Оплата позже»** → SnackBar, **без** backend, **без** загрузки фото, **без** оплаты.

Примеры карточек: Студийный / Деловой / Домашний портрет (free); Премиум, Зимняя, Городской, Вечерний, Портрет в путешествии (100 ₽).

## 6. Gallery (Галерея)

Раздел **«Галерея»** — пока **empty state**, не библиотека файлов.

**Реализовано:**

- Заголовок: «Здесь будут ваши созданные изображения»
- Карточка: иконка `photo_library`, **«Пока нет изображений»**, поясняющий текст
- CTA **«Создать первое изображение»** → переключение на вкладку **Создать**
- Блок **«Скоро»**: после аккаунта появится история изображений и фотосессий

**Позже:** сетка/список, скачать, повторить стиль, удалить, «создать похожее» (публичный API истории).

---

## 7. Packs (Пакеты)

Вкладка **«Пакеты генераций»** — экран покупки **генераций** (custom mode на «Создать»), **без** слова «кредиты» в UI.

**Пакеты (отображаются как количество изображений):**

| Название | Цена | Объём |
|----------|------|--------|
| Стартовый | 199 ₽ | 25 изображений |
| Авторский | 499 ₽ | 100 изображений (badge «Популярный», акцентная карточка) |
| Профи | 1199 ₽ | 250 изображений (badge «Выгодно») |

Карточка **«Всё просто»**: «1 генерация = 1 изображение». Кнопка **«Скоро»** → SnackBar «Оплата будет добавлена позже».

**Responsive layout:** до 3 колонок (широкий web), 2 на средней ширине, 1 на мобильном.

**Позже:** RuStore Billing. Не путать с оплатой **фотосессии** (100 ₽ за набор 3 фото на вкладке «Фотосессии»).

## 8. Profile (Профиль)

Placeholder под **будущую авторизацию** и настройки.

**Реализовано:**

- Карточка **«Вход будет доступен позже»**: после входа сохраняются изображения, фотосессии и пакеты генераций в аккаунте
- Блок **«Что появится здесь»**: созданные изображения, история фотосессий, купленные пакеты, настройки
- **«Безопасность»**: ключи и платежи не в приложении, важные операции на сервере
- Заглушки **«Войти позже»** / **«Политика конфиденциальности»** → SnackBar

**Позже:** Supabase Auth, синхронизация баланса генераций.

---

## 9. Create (Создать)

Вкладка **«Создать»** — пользователь вводит **описание изображения** (не «промпт»).

**Реализовано:**

- Поле описания, placeholder «Например: портрет в деловом стиле»
- **Быстрые идеи** (chips на **русском**): Киберпанк-кот, Уютный дом, Премиум-реклама, Аниме-портрет, Город будущего — вставляют текст в поле
- Блок **«Как получить хороший результат»**: 3 подсказки + пример («Женский деловой портрет…»)
- Кнопка **«Создать изображение»** → `POST /generate` через `ApiService`
- Карточка статуса: в **demo mode** (`ENABLE_CREDIT_CONSUMPTION=false` на backend) показывается «Демо-режим: генерации не списываются»; при включённом списании — остаток бесплатных / купленных генераций
- Результат: preview изображения + «Создано по описанию: …»

**Позже:** переключатель «одно фото / фотосессия 3 фото», загрузка референса.

## 10. MVP scope for UI

### Сделано в текущем Flutter MVP

- [x] Нижняя навигация: **Создать · Фотосессии · Галерея · Пакеты · Профиль** (русский UI)
- [x] **Создать** — `POST /generate`, подсказки, русские chips, demo mode в статусе
- [x] **Фотосессии** — 8 preview-карточек, responsive 1–2 колонки, free / 100 ₽, SnackBar
- [x] **Галерея** — empty state, CTA на «Создать»
- [x] **Пакеты** — 3 пакета (199 / 499 / 1199 ₽), responsive 1–3 колонки, «Скоро»
- [x] **Профиль** — placeholder входа, безопасность, будущие разделы
- [x] UX без «промпт» / «кредиты» / «токены» в видимых текстах

### Не делаем в текущем UI MVP

- Реальная загрузка фото пользователя  
- Watermark / preview vs full quality  
- Настоящая фотосессия из 3 изображений  
- Оплата (RuStore)  
- Сложная библиотека с фильтрами  
- Авторизация

## 11. Future backend implications

Расширения API и БД для шаблонов, фотосессий и загрузок:

| Поле / концепт | Назначение |
|----------------|------------|
| `generation_type` | `single` \| `photoshoot` |
| `template_id` | Связь с каталогом шаблонов |
| `credit_cost` | 1 или 3 (и др.) за операцию |
| `is_watermarked` | Preview vs финальный файл |
| `group_id` | Объединяет 3 фото одной фотосессии |
| `source_image_url` | Фото пользователя для img2img / face |

Таблица `generations` и credit logic потребуют миграций — см. [database_schema.md](database_schema.md).
