# Промпты для генерации preview-картинок

Готовые тексты для **ChatGPT (DALL·E)**, **Gemini Image**, **Midjourney** и других генераторов.  
Цель — подготовить реальные preview для карточек в приложении **AI Image App**.

**Аудитория:** женщины **40+**.  
**Стиль:** красивый, тёплый, аккуратный, реалистичный, «дорогой», без молодёжного визуала, без кривых лиц и без лишней сексуализации.

Связанные документы: [preview_content_plan.md](preview_content_plan.md), [preview_assets_checklist.md](preview_assets_checklist.md).

---

## 1. Общие правила генерации preview

### Этика и модель

- **Не используйте** реальные фото людей без разрешения.
- Для preview лучше генерировать **нейтральную AI-модель** (вымышленный человек), а не копировать знаменитость или знакомого.
- Лицо должно выглядеть **естественно**: возраст примерно **45–58 лет**, спокойная мимика, без «пластиковой» ретуши.

### Качество и запреты

- **Без текста**, логотипов, водяных знаков, рамок, UI-элементов.
- **Реалистичная** фотография, мягкий свет, аккуратная обработка.
- **Без** искажённых лиц, лишних пальцев, размытых глаз, кислотных фильтров, откровенной сексуализации, шок-контента.
- Важные детали (лицо, товар) — **ближе к центру** кадра: карточка обрезает края (`BoxFit.cover`).

### Размеры и форматы

| Тип | Размер | Формат | Папка |
|-----|--------|--------|-------|
| Шаблоны (1 фото) | **1200×800** или **1280×720** | JPG | `frontend/assets/previews/templates/` |
| Фотосессии (серия) | **1024×1024** (квадрат) | JPG | `frontend/assets/previews/photoshoots/` |
| Good / bad guide | **1024×1024** | JPG | `frontend/assets/guides/` |

### Серии фотосессий

Для одной фотосессии нужны **3 кадра** одного человека в **одном стиле**: тот же свет, палитра, одежда, настроение.  
Удобно: сначала сгенерировать **кадр 1**, затем в том же чате попросить «ещё 2 кадра в том же стиле».

### Универсальный negative prompt (можно добавлять ко всем запросам)

```
текст на изображении, надписи, логотип, водяной знак, watermark, рамка, коллаж, сетка из фото,
искажённое лицо, лишние пальцы, лишние руки, размытое лицо, пластиковая кожа, кислотные цвета,
откровенная сексуализация, шок-контент, мультяшный стиль, аниме, 3D render, низкое качество,
шум, артефакты, уродливые пропорции, знаменитость, копия реального человека
```

**English (для Midjourney / некоторых API):**

```
text, logo, watermark, frame, collage, grid, distorted face, extra fingers, extra hands,
blurry face, plastic skin, oversaturated, sexualized, cartoon, anime, 3d render, low quality,
noise, artifacts, bad proportions, celebrity, real person likeness
```

```

---

### `bad_photo.jpg`

**Prompt:**

```
Пример НЕудачного исходного фото для обучающего экрана (не страшное, не уродливое):
женщина 50 лет в тёмной комнате, лицо маленькое и далеко в кадре, слегка размыто,
часть лица в тени, снято снизу, понятно что качество плохое для AI — но человек выглядит нормально

## 5. Как сохранять и добавлять в проект

1. **Сгенерируйте** картинку в ChatGPT / Gemini / другом генераторе с нужным размером.
2. **Скачайте** результат в **JPG** (качество 85–92% — обычно достаточно).
3. **Переименуйте** файл **точно** как в этом документе (регистр и подчёркивания важны).
4. **Положите** в папку:
   - шаблоны → `frontend/assets/previews/templates/`
   - фотосессии → `frontend/assets/previews/photoshoots/`
   - guide → `frontend/assets/guides/`
5. Убедитесь, что папки уже указаны в `frontend/pubspec.yaml` (секция `assets:`).
6. В терминале:

```powershell
cd frontend
flutter clean
flutter pub get
flutter run
```

7. **Проверьте** карточки в приложении: шаблон, фотосессия, экран «Свой запрос».

**Опционально (без пересборки APK):** загрузите JPG в Supabase Storage и укажите `previewUrl` / `previewUrls` в `backend/app/catalog/` — см. [preview_content_plan.md](preview_content_plan.md).

## 6. Чеклист перед добавлением файла

- [ ] Картинка открывается в просмотрщике (не битый JPG)
- [ ] Лицо не искажено (глаза, рот, пальцы в норме)
- [ ] **Нет текста** на картинке
- [ ] **Нет водяного знака** генератора
- [ ] Стиль подходит категории (деловой / нежный / товар и т.д.)
- [ ] Возраст и настроение соответствуют аудитории **40+**
- [ ] Файл назван **правильно** (см. путь в документе)
- [ ] Размер близок к рекомендованному (шаблон 1200×800, фотосессия 1024×1024)
- [ ] Для фотосессии: **3 кадра** выглядят как одна серия
- [ ] В приложении **не показывается цветная заглушка** вместо фото
